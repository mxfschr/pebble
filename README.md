**The problem:** Your AI coding assistant forgets everything between sessions. You manually maintain a `CLAUDE.md` file, but it gets stale. Critical decisions evaporate. Patterns get re-debated. The same bugs get re-discovered.

**Pebble fixes this** without touching your CLAUDE.md. It watches your git commits, queues them for review, and gives your AI assistant MCP tools to build its own persistent memory. No API keys needed — your AI assistant IS the intelligence layer.

## Zero Config. No API Keys. Your CLAUDE.md Is Sacred.

Pebble **never overwrites your CLAUDE.md**. Your rules, workflow, and identity stay exactly as you wrote them. Pebble writes to its own files:

```
CLAUDE.md              ← yours, untouched (Pebble adds one pointer line on init)
soul.md                ← your AI's personality (template provided, you customize)
.pebble/memory.md      ← accumulated knowledge (auto-generated by Pebble)
.pebble/context-tree/  ← detailed memories as markdown files
```

## Quickstart

```bash
npm install -g pebble-memory

cd your-project
pebble init
```

That's it. Pebble will:
- ✅ Install a git post-commit hook
- ✅ Create `.pebble/memory.md` with instructions for Claude Code
- ✅ Add a one-line pointer to your existing CLAUDE.md
- ✅ Create a `soul.md` personality template
- ✅ Set up the MCP server for Claude Code

## The Three-File System

### `CLAUDE.md` — Your Rules (Manual)
Your workflow, tech stack, behavior rules, infrastructure details. Pebble adds one line on init pointing to memory.md. That's it. Never touched again.

### `soul.md` — AI Personality (Manual)
How your AI assistant should feel. Not rules — personality. Pebble creates a starter template:

```markdown
## Voice
- Direct, no filler
- German for discussions, English for code
- Give real opinions when asked

## Mindset
- We're a team: I decide, you build
- Think along. Flag what I miss.
- Revenue > Perfection. Shipping > Planning.
```

### `.pebble/memory.md` — Knowledge (Auto-Generated)
Structured memories from git commits and MCP tool calls. Categorized, relevance-decayed, always current. Includes unprocessed commits for Claude to review.

## How It Works

```
┌─────────┐     ┌──────────┐     ┌──────────────┐     ┌──────────────────┐
│ You code │ ──> │ git hook  │ ──> │ Queue in     │ ──> │ .pebble/memory.md│
│ & commit │     │ captures  │     │ SQLite       │     │ regenerated      │
└─────────┘     └──────────┘     └──────────────┘     └──────────────────┘
                                                              │
                                                              ▼
                                                       ┌────────────┐
                                                       │ Claude Code│
                                                       │ next start │
                                                       └─────┬──────┘
                                                             │
                                                  ┌──────────┴──────────┐
                                                  │ pebble_remember()   │
                                                  │ pebble_mark_done()  │
                                                  └──────────┬──────────┘
                                                             │
                                                             ▼
                                                       ┌────────────┐
                                                       │ Memories   │
                                                       │ persist    │
                                                       └────────────┘
```

### Five Memory Categories

| Category | Emoji | What It Captures |
|----------|-------|-----------------|
| **Decision** | ⚡ | Architectural choices with rationale |
| **Pattern** | 🔧 | Code conventions, naming rules |
| **Context** | 📋 | Project domain knowledge |
| **Learning** | 💡 | Bugs, pitfalls, gotchas |
| **Todo** | 🎯 | Active work items |

### Context Tree

Every memory also gets written as a human-readable markdown file:

```
.pebble/context-tree/
├── README.md           ← index of all knowledge
├── decisions/README.md
├── patterns/README.md
├── context/README.md
├── learnings/README.md
└── active-work/README.md
```

These files can be git-tracked and shared with your team. The SQLite DB stays private.

## MCP Tools

| Tool | What It Does |
|------|-------------|
| `pebble_remember` | Store a memory with category and tags |
| `pebble_recall` | Search memories by keyword |
| `pebble_forget` | Remove an outdated memory |
| `pebble_status` | Show memory stats + unprocessed count |
| `pebble_mark_processed` | Clear commits after review |

## CLI

```bash
pebble init              # Initialize Pebble + soul.md
pebble capture           # Queue latest commit (git hook does this)
pebble add <cat> <text>  # Manually add a memory
pebble search <query>    # Search memories
pebble forget <id>       # Remove a memory
pebble status            # Show memory stats
pebble generate          # Regenerate memory.md + context tree
pebble hooks install     # Install git hook
pebble hooks uninstall   # Remove git hook
```

## Configuration

`.pebble/config.json`:

```json
{
  "max_claude_md_lines": 150,
  "relevance_decay_days": 30
}
```

## How It's Different

| | Manual CLAUDE.md | ByteRover | Pebble |
|---|---|---|---|
| **Your CLAUDE.md** | You manage it | N/A | Untouched |
| **Cost** | Free | Paid SaaS | Free, no API keys |
| **Storage** | In the file | Cloud | Local SQLite |
| **AI personality** | No | No | soul.md |
| **Updates** | Manual | Auto (API calls) | Auto (zero API calls) |
| **Team sharing** | Copy/paste | Cloud sync | Git (context-tree/) |
